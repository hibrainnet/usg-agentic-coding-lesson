# 실습 4: 서브에이전트 (10분)

## 이번 실습 목표

서브에이전트 패턴을 직접 체험하고, 직접 탐색과의 컨텍스트 사용량 차이를 비교합니다.  
task-manager 프로젝트를 분석 대상으로 사용합니다.

---

## Step 1: 서브에이전트 없이 넓은 탐색 (비교용)

먼저 서브에이전트 없이 task-manager 전체 분석을 직접 요청합니다.

```
"lessons/05-automation-extension/examples/task-manager/ 디렉토리에 있는
모든 파일을 읽고 다음을 정리해줘:
1. 구현된 함수 목록과 역할
2. 각 함수에 테스트가 있는지 여부
3. 테스트가 없는 함수가 있다면 어떤 테스트를 추가해야 할지"
```

응답 후 `/usage`로 컨텍스트 사용량을 확인합니다.

---

## Step 2: 서브에이전트로 같은 작업 위임

`/clear`로 대화를 초기화하고, 같은 작업을 서브에이전트에게 위임합니다.

```
"서브에이전트를 사용해서 탐색해줘:
lessons/05-automation-extension/examples/task-manager/ 디렉토리를 분석하고
1. 구현된 함수 목록과 역할
2. 각 함수에 테스트가 있는지 여부
3. 테스트가 없는 함수가 있다면 어떤 테스트를 추가해야 할지
탐색은 독립 에이전트에게 맡기고 결과만 요약해서 알려줘."
```

다시 `/usage`로 컨텍스트 사용량을 확인하고 Step 1과 비교합니다.

---

## Step 3: context: fork 스킬로 서브에이전트 실행

스킬의 `context: fork`를 사용하면 스킬 자체가 독립 서브에이전트에서 실행됩니다.

에이전트에게 스킬 파일을 만들어달라고 요청합니다.

```
"`.claude/skills/analyze-task-manager/SKILL.md` 파일을 만들어줘.
task-manager 코드베이스를 심층 분석하는 스킬이야.

frontmatter:
- description: task-manager 코드베이스를 분석해서 함수 목록, 테스트 현황, 개선 제안을 정리한다
- context: fork
- agent: Explore

본문에는 lessons/05-automation-extension/examples/task-manager/ 를 분석해서
함수 목록, 테스트 커버리지 현황, 개선 제안을 정리하도록 지시해줘."
```

실행해보기:

```
/analyze-task-manager
```

메인 대화에는 분석 결과 요약만 돌아오는 것을 확인합니다.

---

## Step 4: 실용적인 서브에이전트 활용

프로젝트 전체 구조 분석을 서브에이전트에게 맡깁니다.

```
"서브에이전트를 사용해서 분석해줘:
lessons/ 디렉토리 전체에서 파이썬 파일을 찾고
각 파일에 pytest 테스트가 있는지 없는지,
있다면 테스트 함수가 몇 개인지 표로 정리해줘.
탐색 작업은 서브에이전트에게 위임해줘."
```

---

## 직접 탐색 vs 서브에이전트 비교 정리

| | 직접 탐색 | 서브에이전트 위임 |
|--|--|--|
| **컨텍스트 소비** | 메인 대화에 모두 쌓임 | 서브에이전트에서 처리 후 요약만 반환 |
| **적합한 작업** | 단순하고 짧은 탐색 | 넓고 긴 탐색 (파일 수십 개 이상) |
| **결과 형식** | 전체 과정이 대화에 노출 | 요약된 결과만 메인 대화에 등장 |

---

## 체크리스트

- [ ] 서브에이전트로 같은 탐색을 위임하고 결과를 비교했다

- [ ] `context: fork` 스킬을 만들고 실행해봤다
