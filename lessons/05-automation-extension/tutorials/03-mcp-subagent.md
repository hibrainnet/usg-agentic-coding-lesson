# 실습 3: MCP 서버 연결 & 서브에이전트 (20분)

## 파트 A: MCP 서버 연결 (10분)

### MCP란?

에이전트가 외부 도구(GitHub, 데이터베이스, 파일시스템 등)에 접근할 수 있게 해주는 표준 연결 방식입니다.

이번 실습에서는 **공개 파일시스템 MCP 서버**를 연결해서,
에이전트가 특정 디렉토리의 파일을 직접 조회하게 합니다.

---

### Step 1: Node.js 설치 확인

MCP 서버 대부분이 Node.js 기반입니다.

```bash
node --version   # v18 이상 권장
npm --version
```

없다면 [nodejs.org](https://nodejs.org)에서 설치합니다.

---

### Step 2: filesystem MCP 서버 연결

```bash
claude mcp add filesystem -- npx -y @modelcontextprotocol/server-filesystem ~/Desktop
```

이 명령어는 `~/Desktop` 폴더를 에이전트가 읽고 쓸 수 있는 MCP 서버로 등록합니다.

연결 확인:
```bash
claude mcp list
```

`filesystem`이 목록에 나오면 성공입니다.

---

### Step 3: MCP 도구 사용해보기

Claude Code를 새로 열고(또는 `/reset`) 에이전트에게 요청합니다.

```
"Desktop 폴더에 있는 파일 목록을 보여줘."
```

에이전트가 MCP를 통해 파일 시스템에 직접 접근해서 응답하는 것을 확인합니다.

---

### Step 4: (선택) GitHub MCP 서버

GitHub 계정이 있다면 GitHub MCP 서버도 연결해볼 수 있습니다.

```bash
# GitHub Personal Access Token 필요
claude mcp add github -- npx -y @modelcontextprotocol/server-github
```

연결 후:
```
"내 GitHub 저장소 목록을 보여줘."
"[저장소명]의 최근 이슈를 보여줘."
```

---

## 파트 B: 서브에이전트 패턴 (10분)

### 서브에이전트란?

메인 에이전트가 **독립적인 에이전트를 만들어 큰 탐색 작업을 위임**하는 패턴입니다.  
메인 대화 컨텍스트를 아끼면서 넓은 탐색을 수행할 수 있습니다.

---

### Step 1: 서브에이전트 없이 넓은 탐색 (비교용)

먼저 서브에이전트 없이 넓은 탐색을 직접 요청합니다.

```
"lessons/ 디렉토리 전체에서 파이썬 파일을 모두 찾아서
각 파일에 pytest 테스트가 있는지 없는지 정리해줘."
```

컨텍스트가 얼마나 사용되는지 `/usage`로 확인합니다.

---

### Step 2: 서브에이전트로 같은 작업 위임

같은 작업을 서브에이전트에게 위임합니다.

```
"서브에이전트를 사용해서 탐색해줘:
lessons/ 디렉토리 전체에서 파이썬 파일을 모두 찾고
각 파일에 pytest 테스트가 있는지 없는지 정리해줘.
탐색은 독립 에이전트에게 맡기고 결과만 요약해서 알려줘."
```

`/usage`로 컨텍스트 사용량을 다시 확인하고 차이를 비교합니다.

---

### Step 3: 실용적인 서브에이전트 활용

실제로 유용한 탐색 작업을 서브에이전트에게 맡겨봅니다.

```
"이 프로젝트 전체의 마크다운 파일을 탐색해서
각 교시(01~06)마다 tutorials 파일이 몇 개 있는지 표로 정리해줘.
탐색 작업은 서브에이전트에게 위임해줘."
```

---

## 체크리스트

**MCP**
- [ ] `claude mcp add`로 MCP 서버를 등록했다
- [ ] `claude mcp list`로 등록된 서버를 확인했다
- [ ] 에이전트가 MCP를 통해 외부 데이터에 접근하는 것을 확인했다

**서브에이전트**
- [ ] 서브에이전트 없이 넓은 탐색을 요청해봤다
- [ ] 서브에이전트로 같은 탐색을 위임하고 결과를 비교했다
- [ ] 컨텍스트 사용량(`/usage`) 차이를 확인했다
