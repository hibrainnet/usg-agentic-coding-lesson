# 컨텍스트 관리: CLAUDE.md와 계획 모드

## 컨텍스트란 무엇인가

에이전트가 작업을 잘 하려면 **배경 지식**이 필요합니다. 이 배경 지식을 에이전트에게 전달하는 두 가지 방법이 있습니다.

1. **대화 컨텍스트** — 현재 세션에서 주고받은 대화 내용
2. **CLAUDE.md** — 프로젝트에 대해 에이전트에게 항상 알려주는 파일

2교시에서 배운 `/clear`는 대화 컨텍스트를 초기화합니다. CLAUDE.md는 `/clear` 후에도 남아 있어 **프로젝트의 장기 기억** 역할을 합니다.

---

## 1. CLAUDE.md: 에이전트의 프로젝트 위키

`CLAUDE.md`는 프로젝트 루트에 두는 마크다운 파일입니다. Claude Code를 실행하면 이 파일을 자동으로 읽어 **모든 대화의 시작 컨텍스트**로 사용합니다.

### CLAUDE.md가 없을 때

```
> add_contact에 이메일 검증 추가해줘
Claude: (프로젝트 구조를 모르므로) 파일을 탐색하고 맥락을 파악한 뒤 작업...
        에러 메시지를 영어로 작성...
        테스트는 추가하지 않음...
```

### CLAUDE.md가 있을 때

```
[CLAUDE.md에 "에러 메시지는 한국어, 타입 힌트 필수, 테스트 필수" 명시]

> add_contact에 이메일 검증 추가해줘
Claude: (이미 규칙을 알고 있으므로) 한국어 에러, 타입 힌트, 테스트 자동 추가
```

한 번 잘 써두면 **매번 같은 설명을 반복하지 않아도 됩니다.**

---

## 2. CLAUDE.md 구성 요소

잘 작성된 CLAUDE.md는 다음 항목을 포함합니다.

```markdown
# 프로젝트 이름

## 프로젝트 개요
한 줄 설명.

## 실행 방법
python3 main.py ...          ← 에이전트가 직접 실행할 명령어

## 테스트
pytest tests/ -v             ← 에이전트가 검증할 때 쓰는 명령어

## 코딩 규칙
- 타입 힌트 필수
- 에러 메시지는 한국어
- 새 기능 추가 시 테스트 필수

## 아키텍처
파일/모듈 역할 요약
```

### 항목별 역할

| 항목 | 왜 중요한가 |
|------|------------|
| 실행 방법 | 에이전트가 테스트 결과를 직접 확인할 수 있음 |
| 테스트 명령어 | 수정 후 자동으로 검증할 수 있음 |
| 코딩 규칙 | 매 요청마다 "한국어로 해줘"를 반복하지 않아도 됨 |
| 아키텍처 | 파일 탐색 없이 바로 올바른 파일을 수정함 |

## 3. `/init` — CLAUDE.md 초안 자동 생성

```
> /init
```

현재 프로젝트를 분석해 CLAUDE.md 초안을 만들어줍니다. 자동 생성된 초안을 기반으로 **팀·프로젝트만의 규칙을 직접 추가**해야 합니다. 결과물은 시작점이지 완성본이 아닙니다.

---

## 4. 대화 컨텍스트 관리

대화 컨텍스트가 쌓일수록 토큰이 늘고 응답이 느려질 수 있습니다. 이번 시간 실습에서 이미 `/compact`를 써봤지만, 둘의 차이를 다시 정리합니다.

| 커맨드 | 언제 | 효과 |
|--------|------|------|
| `/clear` | 작업이 바뀔 때 | 대화 초기화, CLAUDE.md는 유지 |
| `/compact [지침]` | 같은 작업 중 대화가 길어졌을 때 | 대화를 요약·압축, 맥락 유지 |

`/compact`에 지침을 추가하면 어떤 방향으로 요약할지 에이전트에게 알려줄 수 있습니다.

```
> /compact 이메일 검증 기능 추가 완료. 다음은 검색 기능 개선.
```

## 5. 계획 모드(Plan Mode)

복잡한 작업은 에이전트가 바로 실행하기 전에 **계획을 먼저 보고 검토하는 것**이 안전합니다.

### 계획 모드 활성화

```
> /plan
```

또는 작업과 함께 입력:
```
> /plan contact-book에 즐겨찾기 기능 추가
```

또는 요청에 명시:
```
> 구현 전에 계획만 먼저 설명해줘. 내가 승인하면 실행해줘.
```

### 계획 모드에서 할 수 있는 것

```
에이전트: 다음과 같이 진행하겠습니다.
1. contacts.py에 validate_email() 함수 추가
2. add_contact()에서 validate_email() 호출
3. test_contacts.py에 테스트 3건 추가
4. pytest로 검증

진행할까요?

> 3번에서 테스트는 5건으로 늘려줘. 나머지는 OK.

에이전트: 수정된 계획으로 진행하겠습니다.
```

**실행 전에 방향을 바꾸는 것이 실행 후보다 훨씬 쉽습니다.**

### 언제 계획 모드를 쓰나요?

- 여러 파일을 동시에 수정해야 할 때
- 되돌리기 어려운 변경(DB 스키마, API 변경 등)
- 요구사항이 복잡해서 단계를 미리 확인하고 싶을 때

## 📌 이번 시간 도구 요약

| 도구 | 언제 | 효과 |
|------|------|------|
| CLAUDE.md | 프로젝트 시작 시 한 번 작성 | 규칙·맥락을 매번 말하지 않아도 됨 |
| `/init` | CLAUDE.md 첫 작성 시 | 초안 자동 생성 |
| `/compact` | 같은 작업 중 대화가 길어졌을 때 | 맥락 유지하며 압축 |
| `/plan` | 복잡하거나 되돌리기 어려운 작업 전 | 실행 전 방향 검토 |
