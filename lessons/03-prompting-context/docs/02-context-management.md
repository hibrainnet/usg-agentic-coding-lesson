# 컨텍스트 관리: CLAUDE.md와 계획 모드

## 컨텍스트란 무엇인가

에이전트가 작업을 잘 하려면 **배경 지식**이 필요합니다. 이 배경 지식을 에이전트에게 전달하는 방법이 두 가지입니다.

1. **대화 컨텍스트** — 현재 세션에서 주고받은 대화 내용
2. **CLAUDE.md** — 프로젝트에 대해 에이전트에게 항상 알려주는 파일

---

## 1. CLAUDE.md: 에이전트의 팀 위키

`CLAUDE.md`는 프로젝트 루트에 놓는 마크다운 파일입니다. Claude Code를 실행하면 이 파일을 자동으로 읽어 **모든 대화의 시작 컨텍스트**로 사용합니다.

### CLAUDE.md가 없을 때

```
> add_contact에 이메일 검증 추가해줘
Claude: (프로젝트 구조를 모르므로) 파일을 탐색하고 맥락을 파악한 뒤 작업...
        에러 메시지를 영어로 작성...
        테스트는 영어로 작성...
```

### CLAUDE.md가 있을 때

```
[CLAUDE.md에 "에러 메시지는 한국어, 타입 힌트 필수, 테스트 필수" 명시]

> add_contact에 이메일 검증 추가해줘
Claude: (이미 규칙을 알고 있으므로) 한국어 에러, 타입 힌트 포함, 테스트 자동 추가
```

CLAUDE.md를 잘 쓰면 **매번 같은 설명을 반복하지 않아도 됩니다.**

---

## 2. CLAUDE.md 구성 요소

잘 작성된 CLAUDE.md는 다음 항목을 포함합니다.

```markdown
# 프로젝트 이름

## 프로젝트 개요
한 줄 설명. 에이전트가 전체 맥락을 파악하는 데 쓰인다.

## 실행 방법
python3 main.py ...          ← 에이전트가 직접 실행할 명령어

## 테스트
pytest tests/ -v             ← 에이전트가 검증할 때 쓰는 명령어

## 코딩 규칙
- 타입 힌트 필수             ← 매번 말하지 않아도 되는 규칙
- 에러 메시지는 한국어
- 새 기능 추가 시 테스트 필수

## 아키텍처
파일/모듈 역할 요약          ← 어디에 무엇이 있는지
```

### 항목별 역할

| 항목 | 왜 중요한가 |
|------|------------|
| 실행 방법 | 에이전트가 테스트 결과를 직접 확인할 수 있음 |
| 테스트 명령어 | 수정 후 자동으로 검증할 수 있음 |
| 코딩 규칙 | 매 요청마다 "한국어로 해줘"를 반복하지 않아도 됨 |
| 아키텍처 | 파일 탐색 없이 바로 올바른 파일을 수정함 |

---

## 3. CLAUDE.md 생성하기: `/init`

Claude Code에서 `/init`을 실행하면 현재 프로젝트를 분석해 CLAUDE.md 초안을 자동 생성합니다.

```text
> /init
```

자동 생성된 초안을 기반으로 **팀/프로젝트에 맞는 규칙을 직접 추가**해야 합니다. `/init`의 결과물은 시작점이지, 완성본이 아닙니다.

---

## 4. 대화 컨텍스트 관리

대화가 길어지면 에이전트의 "기억"도 복잡해져서 성능이 떨어집니다. 적절히 정리하세요.

### `/clear` — 새 작업 시작 시

이전 대화의 맥락을 모두 지우고 새로 시작합니다. CLAUDE.md는 유지됩니다.

```text
> /clear
```

언제 쓰나요?
- 이전 작업이 끝나고 완전히 새로운 작업을 시작할 때
- 이전 대화가 현재 작업에 혼동을 줄 수 있을 때

### `/compact` — 대화 요약 압축

현재 대화를 요약해서 맥락을 유지하면서 토큰을 절약합니다.

```text
> /compact
```

언제 쓰나요?
- 같은 작업을 이어가는데 대화가 너무 길어졌을 때

> 🔑 작업 단위로 `/clear`를 습관화하세요. "새 작업 = `/clear`부터 시작"이 좋은 패턴입니다.

## 5. 계획 모드(Plan Mode)

복잡한 작업은 에이전트가 바로 실행하기 전에 **계획을 먼저 보고 검토하는 것**이 안전합니다.

### 계획 모드 활성화

방법 1 — Shift+Tab으로 토글:
```
(대화 입력란에서 Shift+Tab 누르면 계획 모드 아이콘 표시)
```

방법 2 — 요청에 명시:
```text
> 구현하기 전에 계획을 먼저 설명해줘. 승인하면 그때 실행해줘.
> (요청 내용)
```

### 계획 모드에서 할 수 있는 것

```text
에이전트: 다음과 같이 진행하겠습니다.
1. contacts.py에 validate_email() 함수 추가
2. add_contact()에서 validate_email() 호출
3. test_contacts.py에 테스트 3건 추가
4. pytest로 검증

진행할까요?

> 3번에서 테스트는 5건으로 늘려줘. 나머지는 OK.

에이전트: 알겠습니다. 수정된 계획으로 진행하겠습니다.
```

계획 단계에서 수정 요청이 가능합니다. **실행 전에 방향을 바꾸는 것이 실행 후보다 훨씬 쉽습니다.**

### 언제 계획 모드를 쓰나요?

- 여러 파일을 동시에 수정해야 할 때
- 되돌리기 어려운 변경(DB 스키마, API 변경 등)을 할 때
- 요구사항이 복잡해서 단계를 미리 확인하고 싶을 때
- 에이전트가 잘못된 방향으로 가는 것을 사전에 막고 싶을 때

## 📌 정리

| 도구 | 언제 | 효과 |
|------|------|------|
| CLAUDE.md | 항상 (프로젝트 시작 시 작성) | 규칙·맥락을 매번 말하지 않아도 됨 |
| `/init` | CLAUDE.md 첫 작성 시 | 초안 자동 생성 |
| `/clear` | 새 작업 시작 시 | 이전 대화 맥락 정리 |
| `/compact` | 긴 대화 중간 | 맥락 유지하며 압축 |
| 계획 모드 | 복잡하거나 위험한 작업 전 | 실행 전 방향 검토 가능 |
