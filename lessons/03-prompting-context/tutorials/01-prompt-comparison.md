# [실습] 나쁜 프롬프트 vs 좋은 프롬프트 비교

> 🎯 목표: 동일한 작업을 나쁜 프롬프트와 좋은 프롬프트로 각각 요청해서 결과 차이를 직접 확인한다.
> ⏱️ 예상 시간: 15분
> 💡 `examples/contact-book/` 프로젝트를 사용합니다.

---

## 준비

실습용 프로젝트 폴더로 이동합니다.

```bash
cd 경로/lessons/03-prompting-context/examples/contact-book
```

기본 동작을 확인합니다.

```bash
python3 contacts.py add "홍길동" "010-1234-5678"
python3 contacts.py add "김철수" "010-9999-0000" "kim@example.com"
python3 contacts.py list
```

---

## ROUND 1: 나쁜 프롬프트

Claude Code를 실행합니다.

```bash
claude
```

아래 요청을 입력합니다.

```
> 이메일 검증 추가해줘
```

### 관찰하기

Claude가 무언가를 하긴 합니다. 하지만 다음을 확인해 보세요.

- 어느 파일을 수정했나요?
- 이메일이 없는 경우(`""`)도 처리하나요?
- 에러 메시지는 어떻게 생겼나요?
- 테스트를 추가했나요?

에이전트가 **선의로 결정한 것들**이 실제 원하는 것과 다를 수 있습니다.

> 실습 후 `/clear`로 이전 대화 맥락을 초기화합니다.

```
> /clear
```

## ROUND 2: 좋은 프롬프트

이번에는 5가지 구성 요소를 모두 담아서 요청합니다.

```
> contacts.py의 add_contact() 함수에 이메일 형식 검증을 추가해줘.

> 조건:
> - 이메일에 '@'가 없으면 ValueError("올바르지 않은 이메일 형식입니다: {입력값}") 발생
> - 빈 문자열("")은 이메일 없음으로 허용 (검증 생략)
> - 검증 로직은 validate_email(email: str) -> None 함수로 분리해서 contacts.py에 추가
> - 에러 메시지는 한국어로
>
> 범위:
> - contacts.py만 수정할 것
> - storage.py는 건드리지 말 것
>
> 검증:
> - test_contacts.py에 유효한 이메일 2건, 무효한 이메일 2건 테스트 추가 후 pytest 실행
```

### 비교 체크리스트

| 항목 | 나쁜 프롬프트 결과 | 좋은 프롬프트 결과 |
|------|----------|----------|
| 수정된 파일 | ? | contacts.py, test_contacts.py |
| 빈 이메일 처리 | ? | 허용 (검증 생략) |
| 에러 메시지 형식 | ? | 한국어, 입력값 포함 |
| 함수 분리 | ? | validate_email() 별도 함수 |
| 테스트 | ? | 4건 추가 + pytest 통과 |
