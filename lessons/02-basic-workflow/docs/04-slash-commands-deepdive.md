# 슬래시 커맨드 심화 가이드

> 강의 시간이 충분할 때 강사와 함께, 또는 혼자 실습하면서 읽을 수 있도록 설계했습니다.
> 각 커맨드마다 "혼자 해보기" 실습이 있습니다.

> ⚠️ **버전 주의**: 이 문서는 최신 Claude Code 기준으로 작성됐습니다.
> 일부 커맨드는 버전에 따라 동작이 다를 수 있습니다. 항상 `/help`로 실제 설치된 커맨드를 먼저 확인하세요.

## 2. 세션 관리

---

### `/clear` — 대화 맥락 초기화

**역할**: 현재 대화 기록을 모두 지웁니다. CLAUDE.md·설정·권한은 유지됩니다.

```
> /clear
```

별칭: `/reset`, `/new`

**clear vs compact 비교**:

| 구분 | /clear | /compact |
|------|--------|---------|
| 이전 대화 | 완전 삭제 | 요약해서 보존 |
| 맥락 유지 | ❌ | ✅ |
| 언제 | 새 작업 시작 | 같은 작업 계속 |

**언제 쓰나**:
- 작업이 바뀔 때마다 (이것이 가장 중요한 사용처)
- 에이전트가 이전 대화 맥락에 혼동하는 것 같을 때

**혼자 해보기**:
```
1. > todo.py의 add_todo를 설명해줘
2. > /clear
3. > 방금 내가 뭘 물어봤는지 기억해?   ← 기억 못함 확인
```

---

### `/compact` — 대화 압축

**역할**: 긴 대화를 AI가 요약해서 압축합니다. 맥락은 유지하면서 토큰을 절약합니다.

```
> /compact
```

또는 압축 방향을 직접 지정:
```
> /compact complete_todo 버그 수정 중. complete_todo 완료. 다음은 테스트 작성.
```

**언제 쓰나**:
- 같은 작업이 이어지는데 대화가 너무 길어졌을 때
- `/usage`로 확인했을 때 토큰이 많이 쌓였을 때

**혼자 해보기**:
```
1. 여러 번 대화 나누기
2. > /usage    ← 토큰 사용량 확인
3. > /compact
4. > /usage    ← 줄어든 것 확인
5. > 아까 이야기한 내용 기억해?  ← 맥락 유지 확인
```

---

### `/resume` — 이전 대화 재개

**역할**: 저장된 대화 목록을 보여주고 선택해서 이어서 시작합니다.

```
> /resume
```

또는 세션 이름/ID 직접 지정:
```
> /resume todo-cli-작업
```

별칭: `/continue`

**CLI에서도 사용 가능**:
```bash
claude --resume    # 또는  claude -r
```

---

### `/branch` — 대화 분기

**역할**: 현재 대화 지점에서 새로운 분기를 만듭니다. 다른 접근법을 시도해보고 싶을 때 유용합니다.

```
> /branch 우선순위기능-접근법2
```

별칭: `/fork`

**언제 쓰나**:
- "이 방향이 맞는지 잘 모르겠는데 시도해보고 싶을 때"
- 같은 문제에 대한 두 가지 해결책을 비교할 때

---

### `/rewind` — 이전 지점으로 되돌리기

**역할**: 대화를 이전 지점으로 되돌립니다. 에이전트의 마지막 응답을 취소하고 싶을 때 씁니다.

```
> /rewind
```

별칭: `/checkpoint`, `/undo`

**언제 쓰나**:
- 에이전트가 잘못된 방향으로 코드를 수정했을 때
- 마지막 응답이 마음에 들지 않아 다시 시도하고 싶을 때

---

### `/exit` · `/quit` — 세션 종료

```
> /exit    또는   > /quit
```

종료해도 대화 기록은 로컬에 저장됩니다. `claude --continue`로 나중에 이어서 시작할 수 있습니다.

---

## 3. 프로젝트 설정

---

### `/init` — CLAUDE.md 자동 생성

**역할**: 현재 디렉토리를 분석해서 `CLAUDE.md` 초안을 자동으로 작성합니다.

```
> /init
```

**실행하면 Claude가 하는 일**:
```
① 파일 구조·기술 스택 파악
② 주요 파일 읽기 (requirements.txt, package.json 등)
③ 실행 명령어·테스트 명령어 감지
④ CLAUDE.md 초안 작성
⑤ 파일 생성 전 미리보기 → 허용 여부 확인
```

> ⚠️ `/init` 결과물은 시작점입니다. 반드시 검토하고 팀·프로젝트만의 규칙을 직접 추가하세요.

**직접 추가해야 할 항목**:
```markdown
## 코딩 규칙
- 에러 메시지는 한국어로
- 타입 힌트 필수
- 새 기능은 테스트 포함

## 주의사항
- API 키는 코드에 하드코딩 금지
- DB 스키마 변경 시 팀 공유
```

**혼자 해보기**:
```
1. examples/todo-cli/ 이동
2. > /init
3. 생성된 CLAUDE.md 확인
4. 코딩 규칙 섹션 직접 추가
5. > /clear
6. 규칙 명시 없이 기능 추가 요청
7. 규칙이 자동 적용됐는지 확인
```

---

### `/memory` — CLAUDE.md 편집·자동 메모리 관리

**역할**: 프로젝트의 CLAUDE.md 파일을 열어서 편집하거나, 자동 메모리 기능을 켜고 끕니다.

```
> /memory
```

자동 메모리 기능이 활성화되면 Claude가 중요한 컨텍스트를 스스로 CLAUDE.md에 기록합니다.

---

### `/permissions` — 도구 허용·차단 규칙

**역할**: 어떤 도구(파일 쓰기, 명령 실행 등)를 자동 허용하거나 항상 물어보거나 차단할지 설정합니다.

```
> /permissions
```

별칭: `/allowed-tools`

---

### `/hooks` — 훅 설정 확인

**역할**: 특정 이벤트(도구 실행 전/후 등)에 자동으로 실행되는 셸 명령 목록을 봅니다.

```
> /hooks
```

훅 설정은 `.claude/settings.json`에서 합니다. (5교시에서 실습)

## 4. 개발 워크플로우

---

### `/plan` — 계획 모드

**역할**: 에이전트가 바로 실행하지 않고 먼저 계획을 설명합니다. 검토 후 진행 여부를 결정할 수 있습니다.

```
> /plan
(계획 모드 진입 후 작업 입력)
```

또는 작업을 함께 입력:
```
> /plan todo.py에 우선순위 필드 추가
```

**방법 2 — 요청에 명시**:
```
> 구현 전에 계획만 먼저 설명해줘. 내가 승인하면 실행해줘.
```

**언제 쓰나**:
- 여러 파일을 동시에 수정해야 할 때
- 되돌리기 어려운 변경을 할 때
- 에이전트가 어떤 방향으로 갈지 미리 확인하고 싶을 때

---

### `/model` — 모델 확인·변경

**역할**: 현재 모델을 확인하거나 다른 Claude 모델로 전환합니다.

```
> /model
```

인수 없이 실행하면 인터랙티브 선택 화면이 표시됩니다.

**모델 계열 비교**:

| 계열 | 특징 | 적합한 상황 |
|------|------|------------|
| **Opus** | 최고 품질·느림·비쌈 | 복잡한 설계, 어려운 버그 분석 |
| **Sonnet** | 균형 품질·속도·비용 | 일반적인 개발 작업 (기본값) |
| **Haiku** | 빠름·저렴 | 단순 코드 설명, 짧은 반복 작업 |

---

### `/effort` — 추론 노력 수준

**역할**: 모델이 얼마나 깊이 생각할지 설정합니다. 높을수록 정확하지만 느리고 비쌉니다.

```
> /effort
```

수준: `low` → `medium` → `high` → `xhigh` → `max` → `ultracode` → `auto`

---

### `/context` — 컨텍스트 사용량 시각화

**역할**: 현재 컨텍스트 윈도우가 얼마나 차 있는지 시각적으로 보여줍니다.

```
> /context
```

최적화 제안도 함께 표시됩니다. 컨텍스트가 많이 차 있다면 `/compact` 또는 `/clear`를 고려하세요.

---

### `/diff` — 인터랙티브 diff 뷰어

**역할**: 변경된 파일을 보기 좋게 보여주는 diff 뷰어를 엽니다.

```
> /diff
```

방향키로 변경 내용을 탐색할 수 있습니다.

---

### `/btw` — 맥락을 오염시키지 않는 질문

**역할**: 대화 기록에 남기지 않고 간단한 질문을 합니다. 현재 작업 맥락을 방해하지 않습니다.

```
> /btw Python에서 @property가 뭐야?
```

잠깐 궁금한 것을 물어보되, 그 대화가 본 작업 맥락에 쌓이지 않게 하고 싶을 때 유용합니다.

## 5. 코드 품질 검토

---

### `/code-review` — 코드 리뷰

**역할**: 변경된 코드를 다양한 깊이로 리뷰합니다.

```
> /code-review
> /code-review high
> /code-review ultra
```

**레벨**:

| 레벨 | 내용 |
|------|------|
| `low` | 명확한 버그만 |
| `medium` | 버그 + 간단한 개선점 |
| `high` | 버그 + 개선 + 설계 검토 |
| `xhigh` / `max` | 더 넓은 커버리지 |
| `ultra` | 클라우드 다중 에이전트 심층 리뷰 |

플래그:
- `--fix`: 발견된 문제를 자동 수정
- `--comment`: PR 인라인 코멘트로 게시

---

### `/simplify` — 코드 정리

**역할**: 코드의 가독성·효율·중복 제거를 위한 정리를 제안하고 적용합니다. 버그 수정은 하지 않습니다.

```
> /simplify
> /simplify todo.py
```

---

### `/security-review` — 보안 분석

**역할**: 변경된 코드에서 보안 취약점(SQL 인젝션, 인증 문제, 민감 데이터 노출 등)을 분석합니다.

```
> /security-review
```

---

## 6. 정보·상태 확인

---

### `/help` — 전체 도움말

```
> /help
```

실제 설치된 버전의 커맨드 목록이 표시됩니다. 새 버전에서 추가된 커맨드를 확인하려면 항상 `/help`를 먼저 실행하세요.

---

### `/usage` — 토큰 사용량·비용

**역할**: 현재 세션에서 사용한 토큰과 예상 비용을 보여줍니다.

```
> /usage
```

별칭: `/cost`, `/stats`

**출력 예시**:
```
Session cost:
  Input tokens:   12,450
  Output tokens:   3,210
  Cache tokens:    8,900
  
  Estimated cost: $0.08
```

**토큰이란?**
- 한글 1글자 ≈ 1~2 토큰
- 영어 단어 ≈ 1 토큰
- 코드 한 줄 ≈ 5~15 토큰

---

### `/status` — 계정·연결 상태

**역할**: 모델, 계정, API 연결 상태를 한눈에 보여줍니다. 작업 중에도 응답을 방해하지 않고 실행됩니다.

```
> /status
```

---

### `/doctor` — 설치 환경 진단

**역할**: Claude Code 설치 상태, Node.js 호환성, 연결 상태 등을 진단합니다. `f`를 누르면 자동 수정을 시도합니다.

```
> /doctor
```

**언제 쓰나**: 설치 후 검증, 이상한 오류가 반복될 때, 업데이트 후 문제가 생겼을 때

## 8. `claude` CLI 플래그

슬래시 커맨드는 세션 안에서, CLI 플래그는 `claude` 실행 시 붙입니다.

### 기본 실행

```bash
claude                        # 빈 대화형 세션
claude "프롬프트"              # 첫 메시지 포함 시작
```

### 이전 대화 재개

```bash
claude --continue             # 가장 최근 대화 이어서
claude -c

claude --resume               # 대화 목록 보여주고 선택
claude -r
```

### 비대화형 실행

응답만 출력하고 종료. 셸 스크립트 자동화에 유용합니다.

```bash
claude --print "프롬프트"
claude -p "프롬프트"

# 파일 내용 파이프
cat todo.py | claude -p "이 코드의 문제점을 찾아줘"

# 결과 파일 저장
claude -p "todo.py 문서 초안 작성" > docs/todo.md

# 여러 파일 처리 스크립트
for f in *.py; do
  claude -p "$f의 함수 목록을 한 줄씩 출력해줘" >> functions.txt
done
```

### 작업 디렉토리 추가

```bash
claude --add-dir ../shared-utils --add-dir ../api-client
```

Claude가 현재 디렉토리 외에 추가 경로도 참조합니다.

### 모델 지정

```bash
claude --model claude-opus-4-8
claude --model claude-haiku-4-5-20251001
```

### 마크다운 렌더링 끄기

```bash
claude --no-markdown -p "결과를 순수 텍스트로"
```

## 9. 권한(Permission) 시스템 완전 이해

### 권한을 묻는 두 가지 상황

**파일 수정 시**:
```diff
todo.py를 수정하려고 합니다:

-     todos.append({"text": text, "done": False})
+     todos.append({"text": text, "done": False, "priority": 2})

허용하시겠습니까?
❯ 1. 예 — 이번만
  2. 예 — 이 파일 앞으로 항상
  3. 아니오 — 거부
  4. 전체 계획 보기
```

**명령 실행 시**:
```
다음을 실행하려고 합니다:
  $ pytest test_todo.py -v

실행하시겠습니까?
❯ 1. 예 — 이번만
  2. 예 — 이 명령 앞으로 항상
  3. 아니오 — 거부
```

### 선택지 해설

| 선택지 | 효과 | 언제 |
|--------|------|------|
| 이번만 허용 | 한 번만 | 낯선 변경, 처음 보는 파일 |
| 이 파일 앞으로 허용 | 같은 파일 자동 승인 | 자주 수정하는 파일 |
| 이 명령 앞으로 허용 | 같은 명령 자동 승인 | pytest처럼 자주 쓰는 명령 |
| 거부 | 에이전트가 다른 방법 탐색 | 방향이 잘못됐을 때 |

### ESC — 즉시 중단

```
(에이전트 작업 중)  →  ESC  →  현재 작업 즉시 중단
```

중단 후 변경된 파일이 있으면 Git으로 되돌립니다:
```bash
git diff          # 무엇이 바뀌었나
git checkout .    # 모든 변경 되돌리기
```
