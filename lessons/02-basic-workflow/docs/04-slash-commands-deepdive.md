# 슬래시 커맨드 심화 가이드

> 이 문서는 강의 시간이 충분할 때 강사와 함께, 또는 혼자 실습하면서 읽어도 이해할 수 있도록 설계했습니다.
> 각 커맨드마다 "혼자 해보기" 실습이 있습니다. 직접 따라하면서 익히세요.

## 이 문서를 활용하는 방법

| 상황 | 활용법 |
|------|--------|
| 강의 시간 내 | 강사가 데모를 보여주는 동안 따라하기 |
| 자습 시간 | 각 섹션을 차례로 읽고 "혼자 해보기" 실행 |
| 나중에 참고 | 목차에서 필요한 커맨드 찾아보기 |

## 1. 빠른 참고 카드

이 표를 인쇄하거나 별도로 저장해두면 나중에 유용합니다.

```
┌─────────────────────────────────────────────────────┐
│  Claude Code 커맨드 빠른 참고                        │
├───────────────┬─────────────────────────────────────┤
│ /help         │ 커맨드 목록 보기                     │
│ /clear        │ 새 작업 시작 (맥락 초기화)            │
│ /compact      │ 대화 압축 (맥락 유지)                │
│ /init         │ CLAUDE.md 자동 생성                  │
│ /cost         │ 이번 세션 비용·토큰 확인              │
│ /status       │ 계정·API 상태 확인                   │
│ /model        │ 현재 모델 확인 또는 변경              │
│ /config       │ 설정 보기·변경                       │
│ /doctor       │ 설치 환경 진단                       │
│ /vim          │ vim 키 바인딩 토글                   │
│ /bug          │ 버그 신고                            │
│ /exit · /quit │ 세션 종료                            │
├───────────────┼─────────────────────────────────────┤
│ --continue -c │ 이전 대화 이어서 시작                 │
│ --resume -r   │ 대화 목록에서 선택 재개               │
│ --print -p    │ 비대화형 실행 (스크립트용)             │
│ --model       │ 시작 시 모델 지정                     │
│ --add-dir     │ 작업 디렉토리 추가                    │
├───────────────┼─────────────────────────────────────┤
│ ESC           │ 실행 중 작업 즉시 중단                │
│ Ctrl+C        │ 강제 종료                            │
└───────────────┴─────────────────────────────────────┘
```

## 2. 세션 관리 커맨드

세션 관리는 Claude Code를 효율적으로 사용하는 핵심입니다. 컨텍스트를 언제 초기화하고 언제 유지할지 정확히 알아야 합니다.

---

### `/clear` — 대화 맥락 초기화

**역할**: 현재 세션의 대화 기록을 모두 지웁니다. CLAUDE.md와 설정은 그대로 유지됩니다.

**사용법**:
```
> /clear
```

**실행 후 상태**:
```
이전 대화: ████████████████  ← 모두 사라짐
CLAUDE.md:  ████████████████  ← 유지됨
현재 디렉토리: 유지됨
권한 설정:    유지됨
```

**언제 사용하나**:
- 새로운 작업을 시작할 때 (이게 가장 중요한 사용처)
- 에이전트가 이전 대화 때문에 혼동하는 것 같을 때
- 여러 작업을 연속으로 할 때 작업 사이마다

**언제 사용하면 안 되나**:
- 같은 작업을 이어가는 중간에 (맥락이 사라지면 처음부터 다시 설명해야 함)
- `/compact`로 해결할 수 있는 상황 (압축으로 충분할 때)

**혼자 해보기**:
```
1. claude 실행
2. > 파이썬에서 리스트 컴프리헨션이 뭐야?  ← 아무 질문이나 하기
3. 대화가 쌓인 것을 확인
4. > /clear
5. > 방금 내가 뭘 물어봤는지 기억해?      ← 기억 못 함을 확인
```

---

### `/compact` — 대화 압축

**역할**: 긴 대화를 AI가 스스로 요약해서 압축합니다. 맥락은 유지하면서 토큰(비용)을 절약합니다.

**사용법**:
```
> /compact
```

또는 압축 방향을 직접 지정:
```
> /compact 인증 모듈 리팩터링 진행 중. validate_email 추가 완료. 다음은 테스트 작성.
```

**`/clear`와 `/compact`의 차이**:

| 구분 | /clear | /compact |
|------|--------|---------|
| 이전 대화 | 완전히 삭제 | 요약해서 보존 |
| 맥락 유지 | ❌ | ✅ |
| 토큰 절약 | 최대 | 적당히 |
| 언제 | 새 작업 시작 | 같은 작업 계속 |

**언제 사용하나**:
- 한 작업이 길어져서 응답이 느려질 때
- 대화가 길어졌지만 현재 맥락을 잃고 싶지 않을 때
- `/cost`로 확인했더니 토큰이 많이 쌓였을 때

**혼자 해보기**:
```
1. claude 실행
2. 여러 번 대화를 나눔 (예: 코드 설명 요청 여러 번)
3. > /cost  ← 토큰 사용량 확인
4. > /compact
5. > /cost  ← 줄어든 것 확인
6. 이전 맥락이 요약되어 유지됨을 확인
```

---

### `/exit` · `/quit` — 세션 종료

**역할**: Claude Code 세션을 종료합니다. 둘은 완전히 동일합니다.

**사용법**:
```
> /exit
```
또는
```
> /quit
```

또는 키보드로:
```
Ctrl+C   ← 즉시 강제 종료
```

**세션 종료 vs 대화 기록**:
- `/exit`로 종료해도 대화 기록은 로컬에 저장됩니다.
- `claude --continue`로 나중에 이어서 시작할 수 있습니다.

**혼자 해보기**:
```
1. claude 실행
2. > 안녕!
3. > /exit
4. claude --continue  ← 바로 재시작
5. > 방금 뭐라고 했는지 기억해?  ← 기억함을 확인
```

## 3. 프로젝트 설정 커맨드

---

### `/init` — CLAUDE.md 자동 생성

**역할**: 현재 디렉토리의 코드를 분석해서 `CLAUDE.md` 초안을 자동으로 작성합니다.

**사용법**:
```
> /init
```

**실행하면 Claude가 하는 일**:
```
① 현재 디렉토리의 파일 구조 파악
② 주요 파일 내용 읽기 (README, package.json, requirements.txt 등)
③ 프로젝트 목적·기술 스택 파악
④ CLAUDE.md 초안 작성
⑤ 파일 생성 전 내용 미리보기
```

**중요**: `/init`의 결과물은 시작점이지 완성본이 아닙니다. 반드시 검토하고 팀/프로젝트만의 규칙을 추가하세요.

**생성되는 CLAUDE.md 기본 구조**:
```markdown
# 프로젝트명

## 개요
(자동 생성된 설명)

## 실행 방법
(감지된 실행 명령어)

## 테스트
(감지된 테스트 명령어)

## 프로젝트 구조
(파일 역할 설명)
```

**직접 추가해야 할 것들**:
```markdown
## 코딩 규칙
- 에러 메시지는 한국어로
- 타입 힌트 필수
- 새 기능은 반드시 테스트 포함

## 주의사항
- DB 스키마 변경 시 팀 알림 필요
- API 키는 .env에만, 코드에 절대 하드코딩 금지
```

**혼자 해보기**:
```
1. examples/contact-book/ 폴더로 이동
2. claude 실행
3. > /init
4. 생성된 CLAUDE.md 내용 확인
5. code CLAUDE.md 로 열어서 코딩 규칙 섹션 추가
6. > /clear
7. 규칙을 명시하지 않고 간단한 기능 추가 요청
8. CLAUDE.md 규칙이 자동 적용됐는지 확인
```

## 4. 정보·상태 확인 커맨드

---

### `/help` — 전체 도움말

**역할**: 사용 가능한 모든 커맨드 목록과 설명을 출력합니다.

**사용법**:
```
> /help
```

**출력 내용**:
- 내장 슬래시 커맨드 목록
- 각 커맨드의 간단한 설명
- 커스텀 커맨드가 있으면 함께 표시

**활용 팁**: 이 가이드를 읽기 전에 `/help`를 먼저 실행해보는 것이 좋습니다. Claude Code 버전에 따라 커맨드가 달라질 수 있기 때문에, 실제 설치된 버전에서 어떤 커맨드가 있는지 확인하는 습관을 들이세요.

**혼자 해보기**:
```
> /help
← 출력을 이 문서의 목록과 비교해보기
```

---

### `/cost` — 토큰 사용량·비용 확인

**역할**: 현재 세션에서 사용한 토큰 수와 예상 비용을 보여줍니다.

**사용법**:
```
> /cost
```

**출력 예시**:
```
Session cost:
  Input tokens:   12,450
  Output tokens:   3,210
  Cache tokens:   8,900
  
  Estimated cost: $0.08
```

**토큰이란?**
LLM이 텍스트를 처리하는 단위입니다. 대략:
- 한글 1글자 ≈ 1~2 토큰
- 영어 단어 ≈ 1 토큰
- 코드 줄 ≈ 5~15 토큰
- 큰 파일 1개 = 수백~수천 토큰

**언제 사용하나**:
- API 키로 사용 중일 때 비용이 얼마나 쌓였는지 확인
- `/compact` 또는 `/clear` 시점을 결정할 때
- 여러 작업 세션 후 총 비용 파악

**혼자 해보기**:
```
1. claude 실행
2. > /cost  ← 초기 비용 (거의 0)
3. 파일 설명 요청 3~4번
4. > /cost  ← 사용량이 늘어난 것 확인
5. > /compact
6. > /cost  ← 압축 후 비용 변화 확인
```

---

### `/status` — 계정·API 상태 확인

**역할**: 현재 연결된 계정 정보와 API 상태를 확인합니다.

**사용법**:
```
> /status
```

**출력 내용**:
- 로그인된 계정 이메일
- 사용 중인 인증 방식 (Claude.ai / API 키)
- API 연결 상태

**언제 사용하나**:
- "인증이 만료됐나?" 의심될 때
- 다른 계정으로 전환했는지 확인할 때
- 처음 설치 후 인증이 제대로 됐는지 검증

**혼자 해보기**:
```
> /status
← 현재 로그인된 계정 정보 확인
```

---

### `/model` — 모델 확인·변경

**역할**: 현재 사용 중인 Claude 모델을 확인하거나 다른 모델로 전환합니다.

**사용법**:
```
> /model
```

현재 사용 가능한 모델 목록과 현재 선택된 모델이 표시됩니다.

**Claude 모델 계열 이해하기**:

| 모델 계열 | 특징 | 적합한 상황 |
|-----------|------|------------|
| **Opus** | 최고 품질, 느림, 비쌈 | 복잡한 아키텍처 설계, 어려운 버그 분석 |
| **Sonnet** | 균형잡힌 품질·속도·비용 | 일반적인 개발 작업 (기본값) |
| **Haiku** | 빠름, 저렴, 단순 작업 | 간단한 코드 설명, 짧은 반복 작업 |

**언제 모델을 바꾸나**:
- 응답이 너무 느리다 → 더 가벼운 모델로
- 복잡한 문제에서 답이 부정확하다 → 더 강력한 모델로
- API 비용을 줄이고 싶다 → 더 저렴한 모델로

**혼자 해보기**:
```
> /model
← 현재 모델 확인
← 사용 가능한 모델 목록 확인
```

## 5. 설정·환경 커맨드

---

### `/config` — 설정 관리

**역할**: Claude Code의 동작 방식을 설정합니다. 권한, 출력 형식, 자동 승인 목록 등을 관리합니다.

**사용법**:
```
> /config
```

**주요 설정 항목**:

| 설정 | 역할 |
|------|------|
| 권한 허용 목록 | 자동으로 승인할 파일/명령 패턴 |
| 출력 형식 | 마크다운 렌더링 여부 |
| 자동 업데이트 | Claude Code 버전 자동 업데이트 여부 |

**설정 파일 위치**:
- 전역 설정: `~/.claude/settings.json`
- 프로젝트 설정: `.claude/settings.json`

**혼자 해보기**:
```
> /config
← 현재 설정 목록 확인
```

---

### `/vim` — vim 키 바인딩 토글

**역할**: 대화 입력란에서 vim 스타일 키 바인딩을 켜거나 끕니다. vim에 익숙한 사용자를 위한 옵션입니다.

**사용법**:
```
> /vim
```

**vim 키 바인딩이란?**
- `Esc` → 일반 모드 (이동)
- `i` → 입력 모드 (글쓰기)
- `hjkl` → 좌하상우 이동
- `dd` → 줄 삭제
- vim을 모른다면 이 커맨드는 사용할 필요가 없습니다.

**혼자 해보기**:
```
> /vim   ← 활성화
(입력창에서 Esc 눌러보기)
> /vim   ← 다시 끄기
```

---

### `/doctor` — 설치 환경 진단

**역할**: Claude Code의 설치 상태와 의존성을 점검합니다. 뭔가 이상하게 느껴질 때 먼저 실행해보는 진단 도구입니다.

**사용법**:
```
> /doctor
```

**확인하는 것들**:
- Claude Code 버전
- Node.js 버전 호환성
- 인터넷 연결 상태
- 설정 파일 유효성
- 필요한 의존성 설치 여부

**언제 사용하나**:
- 설치 후 처음으로 동작을 확인할 때
- 이상한 오류가 반복될 때
- Claude Code 업데이트 후 문제가 생겼을 때

**혼자 해보기**:
```
> /doctor
← 진단 결과 확인
← 모두 정상이면 초록색 체크, 문제 있으면 빨간색 표시
```

---

### `/bug` — 버그 신고

**역할**: Claude Code 자체의 버그나 이상한 동작을 Anthropic에 신고합니다.

**사용법**:
```
> /bug
```

실행하면 GitHub Issues 페이지가 열리거나 신고 양식이 나타납니다.

**언제 사용하나**:
- Claude Code가 충돌(crash)할 때
- 분명히 잘못된 동작을 반복할 때 (프롬프트 문제가 아닌 도구 자체 버그)
- 예상치 못한 에러 메시지가 나올 때

> 참고: 에이전트가 "틀린 답"을 한 것은 버그 신고 대상이 아닙니다. LLM 특성상 오답이 나올 수 있으며, 이는 프롬프트 개선으로 해결합니다.

## 6. `claude` 실행 플래그 (CLI)

슬래시 커맨드는 세션 안에서 쓰지만, 플래그는 `claude`를 실행할 때 붙입니다.

### 기본 실행

```bash
# 빈 대화형 세션 시작
claude

# 첫 메시지를 미리 입력해서 시작
claude "이 프로젝트 구조를 설명해줘"
```

---

### 이전 대화 재개

```bash
# 가장 최근 대화를 이어서 시작
claude --continue
claude -c          # 단축

# 대화 목록을 보여주고 선택해서 재개
claude --resume
claude -r          # 단축
```

**사용 시나리오**:
```bash
# 어제 하다 만 작업 이어서
$ claude --continue

# 지난주 작업을 특정해서 재개
$ claude --resume
  [1] 2026-05-30  contact-book 이메일 검증 추가
  [2] 2026-05-29  todo-cli 버그 수정
  선택: 1
```

---

### 비대화형 실행 (`--print`)

응답만 출력하고 세션을 종료합니다. 셸 스크립트에서 자동화할 때 유용합니다.

```bash
# 기본 사용
claude --print "이 함수를 설명해줘"
claude -p "이 함수를 설명해줘"     # 단축

# 파일 내용을 파이프로 전달
cat auth.py | claude -p "이 코드의 버그를 찾아줘"

# 결과를 파일로 저장
claude -p "contacts.py 문서화 초안 작성" > docs/contacts.md

# 스크립트 내에서 사용
if claude -p "test_contacts.py 테스트가 모두 통과하는지 확인해줘" | grep -q "PASSED"; then
  echo "테스트 통과"
fi
```

---

### 작업 디렉토리 추가 (`--add-dir`)

```bash
# 여러 프로젝트를 함께 참조할 때
claude --add-dir ../shared-utils --add-dir ../api-client
```

Claude가 현재 디렉토리 외에 추가 경로도 탐색할 수 있게 됩니다.

---

### 모델 지정

```bash
# 특정 모델로 시작
claude --model claude-opus-4-8
claude --model claude-haiku-4-5-20251001
```

---

### 마크다운 렌더링 끄기 (`--no-markdown`)

```bash
# 파이프 처리 또는 로그 저장 시 유용
claude --no-markdown -p "코드 리뷰 결과를 텍스트로"
```

## 7. 권한(Permission) 시스템 완전 이해

Claude Code는 **사람이 통제권을 유지**할 수 있도록 설계됐습니다. 이 섹션에서는 권한 시스템을 완전히 이해합니다.

### 권한을 묻는 두 가지 상황

**상황 1: 파일 수정**

```
contacts.py를 수정하려고 합니다:

    def add_contact(name, phone):
-       contacts.append({"name": name, "phone": phone})
+       contacts.append({"name": name, "phone": phone, "email": ""})

허용하시겠습니까?
❯ 1. 예 — 이번만 허용
  2. 예 — 이 파일은 앞으로 항상 허용
  3. 아니오 — 거부
  4. 전체 계획 보기
```

**상황 2: 명령 실행**

```
다음 명령을 실행하려고 합니다:
  $ pytest test_contacts.py -v

실행하시겠습니까?
❯ 1. 예 — 이번만 허용
  2. 예 — 이 명령은 앞으로 항상 허용
  3. 아니오 — 거부
```

---

### 선택지 완전 해설

| 선택지 | 효과 | 언제 |
|--------|------|------|
| **이번만 허용** | 이 한 번만 | 낯선 변경, 처음 보는 파일 |
| **이 파일 앞으로 허용** | 같은 파일은 이후 자동 승인 | 자주 수정하는 파일 |
| **이 명령 앞으로 허용** | 같은 명령은 이후 자동 승인 | pytest처럼 자주 쓰는 명령 |
| **거부** | 에이전트가 다른 방법 찾음 | 방향이 틀렸을 때 |

> ⚠️ "앞으로 허용"을 너무 많이 쌓으면 에이전트가 거의 묻지 않게 됩니다. `/config`에서 초기화할 수 있습니다.

---

### ESC — 즉시 중단

에이전트가 작업하는 중 **잘못된 방향**으로 가고 있다면 즉시 멈출 수 있습니다.

```
(에이전트가 파일을 수정 중...)
ESC  →  현재 작업 즉시 중단, 대화 프롬프트로 돌아옴
```

멈춘 후 수정된 파일이 있으면 Git으로 되돌릴 수 있습니다:
```bash
git diff         # 무엇이 바뀌었나 확인
git checkout .   # 모든 변경 되돌리기
```

---

### 권한 모드별 특성

| 모드 | 권한 확인 | 적합한 상황 |
|------|-----------|------------|
| **기본 모드** | 매번 확인 | 학습 중, 낯선 코드베이스 |
| **누적 허용** | 허용 목록 내 자동 | 익숙한 프로젝트, 반복 작업 |
| **`--dangerously-skip-permissions`** | 전혀 안 함 | 완전 자동화 스크립트 (위험) |

> ⚠️ `--dangerously-skip-permissions`는 CI/CD 등 완전 자동화 환경에서만 사용하세요. 개발 중에는 반드시 확인 모드를 유지하세요.

## 8. 커스텀 슬래시 커맨드 미리보기

5교시에서 자세히 다루지만, 개념만 미리 알아둡니다.

### 개념

`.claude/commands/` 디렉토리에 마크다운 파일을 만들면, 그 파일명이 슬래시 커맨드가 됩니다.

```
프로젝트 루트/
└── .claude/
    └── commands/
        ├── review.md     → /review 커맨드 생성
        ├── commit.md     → /commit 커맨드 생성
        └── test.md       → /test 커맨드 생성
```

### 예시: `/review` 커맨드

파일: `.claude/commands/review.md`

```markdown
변경된 파일을 검토해줘.
다음 항목을 확인해:
1. 버그나 논리 오류
2. 누락된 에러 처리
3. 테스트 커버리지
4. 코딩 컨벤션(CLAUDE.md 기준)

각 항목을 간결하게 bullet로 정리해줘.
```

이제 세션에서:
```
> /review
```
입력하면 위 프롬프트가 자동으로 실행됩니다.

### 전역 커스텀 커맨드

프로젝트에 무관하게 쓰고 싶다면:
```
~/.claude/commands/
├── review.md    → 모든 프로젝트에서 /review 사용 가능
└── standup.md   → 스탠드업 미팅 요약 커맨드
```

## 10. 자주 발생하는 상황과 해결책

### "에이전트가 엉뚱한 방향으로 간다"

```
즉시 ESC → 상황 설명 → 다시 요청
```
또는
```
> /clear → 더 명확한 프롬프트로 재요청
```

---

### "응답이 너무 느리다"

원인 1 — 대화가 너무 김:
```
> /compact  또는  > /clear
```

원인 2 — 모델이 무거움:
```
> /model  (더 가벼운 모델로 변경)
```

원인 3 — 네트워크 문제:
```
> /status  (API 상태 확인)
```

---

### "이전에 쓰던 설정이 초기화됐다"

`/clear`는 대화만 지웁니다. 권한 설정은 `/config`에서 관리합니다.

---

### "어느 계정으로 로그인됐는지 모르겠다"

```
> /status
```

---

### "Claude Code 자체가 이상하게 동작한다"

```
> /doctor    ← 환경 진단
> /bug       ← 버그 신고 (필요시)
```
