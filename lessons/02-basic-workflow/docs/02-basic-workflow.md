# 기본 워크플로우: 탐색·질문·수정·검증

환경 설정이 끝나면 Claude Code와 함께하는 **기본 작업 흐름**을 익힙니다. 이 흐름은 앞으로 6시간 내내 반복하게 됩니다.

---

## 1. 워크플로우의 기본 단위: 대화 한 번

Claude Code와의 협업은 다음 패턴으로 이루어집니다.

```
사람이 요청       →  에이전트가 작업  →  사람이 검토
"이 코드 설명해줘"   파일 읽기·분석      결과 확인·질문
"버그 찾아 고쳐줘"   탐색→계획→수정→검증  변경 내용 승인
```

1교시에서 배운 **탐색→계획→실행→검증 루프**가 한 번의 요청 안에서 자동으로 돕니다. 여러분이 할 일은 요청을 명확히 하고 결과를 검토하는 것입니다.

---

## 2. 세 가지 핵심 요청 유형

### (1) 탐색 요청 — "이게 뭐야?"

코드를 이해하는 것에서 모든 작업이 시작됩니다. Claude Code는 파일을 직접 읽고 분석하므로, 붙여넣기 없이 질문만 하면 됩니다.

```text
> 이 프로젝트의 구조를 설명해줘.
> todo.py에서 complete_todo 함수가 하는 일을 설명해줘.
> 이 코드에서 JSON 파일을 쓰는 부분이 어디야?
```

### (2) 수정 요청 — "이걸 고쳐줘 / 추가해줘"

자연어로 원하는 결과를 설명하면 에이전트가 파일을 직접 수정합니다. 수정 전에 변경 내용 미리보기를 보여주고 허락을 묻습니다.

```text
> complete_todo에 IndexError가 나는 버그가 있는 것 같아. 찾아서 고쳐줘.
> 할 일을 삭제하는 delete 명령을 추가해줘.
> 모든 함수에 한국어 docstring을 추가해줘.
```

### (3) 검증 요청 — "이게 맞는지 확인해줘"

수정 후에는 에이전트가 직접 테스트를 실행하거나, 여러분이 요청해서 실행할 수 있습니다.

```text
> 방금 고친 게 맞는지 테스트를 돌려서 확인해줘.
> pytest를 실행하고 결과를 알려줘.
```

---

## 3. 슬래시 커맨드 — 첫 세션에 꼭 쓰는 것

대화 중 `/`로 시작하는 명령으로 Claude Code를 제어합니다. 지금은 세 가지만 기억하세요.

| 커맨드 | 동작 |
|--------|------|
| `/help` | 사용 가능한 커맨드 목록 보기 |
| `/clear` | 대화 맥락 초기화 — 새 작업 시작 시 |
| `/exit` | 세션 종료 |

> 🔑 작업이 바뀌면 `/clear`를 먼저 실행하세요. 이전 작업의 맥락이 새 작업에 혼동을 줄 수 있습니다.

전체 커맨드 목록과 CLI 플래그는 `docs/03-slash-commands.md`를 참고하세요.
**실전 활용법(언제·왜 쓰는가)은 3교시에서 시나리오와 함께 다룹니다.**

---

## 4. 권한(Permission) 다시 보기

에이전트가 파일을 수정하거나 명령을 실행하기 전에는 항상 허락을 구합니다. 이 시점에 **변경 내용을 반드시 읽어보세요.**

```diff
  todo.py 파일을 수정하려고 합니다:

-     todos[n]["done"] = True
+     todos[n - 1]["done"] = True

  이 변경을 적용할까요?
```

- **허용**: 변경을 적용합니다.
- **거부**: 에이전트가 다른 방법을 제안합니다.
- **ESC**: 진행 중인 작업을 즉시 중단합니다.

> ⚠️ "에이전트가 제안했으니 맞겠지"라는 생각은 금물입니다. 변경 내용을 읽고 이해한 뒤 허용하는 습관이 중요합니다.

---

## 5. 세션 재개

| 상황 | 방법 |
|------|------|
| 새 작업 시작 | `/clear` |
| 세션 종료 | `/exit` 또는 Ctrl+C |
| 이전 대화 이어서 시작 | `claude --continue` |
| 대화 목록에서 선택 재개 | `claude --resume` |

---

## 📌 기본 워크플로우 요약

1. 프로젝트 폴더에서 `claude` 실행
2. 탐색 요청으로 코드 구조 파악
3. 수정 요청 → 변경 미리보기 확인 → 허용
4. 검증 요청 또는 직접 실행으로 결과 확인
5. 만족하면 완료, 아니면 추가 요청

다음은 `tutorials/03-first-workflow.md`에서 이 흐름을 **To-Do CLI** 프로젝트로 직접 실습합니다.
