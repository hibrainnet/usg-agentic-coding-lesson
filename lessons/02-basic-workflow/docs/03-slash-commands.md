# 슬래시 커맨드 & CLI 레퍼런스

Claude Code는 `/`로 시작하는 내장 커맨드와 터미널 실행 시 사용하는 CLI 플래그를 제공합니다. 대화 흐름을 제어하는 핵심 도구이므로 숙지해두면 효율이 크게 높아집니다.

---

## 슬래시 커맨드 전체 목록

대화 입력란에서 `/`를 입력하면 사용 가능한 커맨드 목록이 자동 완성됩니다.

### 세션 관리

| 커맨드 | 동작 | 언제 사용하나 |
|--------|------|--------------|
| `/clear` | 대화 컨텍스트 초기화 (CLAUDE.md는 유지) | 새 작업을 시작할 때, 이전 맥락이 혼동을 줄 때 |
| `/compact` | 긴 대화를 요약·압축 (맥락 유지하며 토큰 절약) | 같은 작업이 이어지는데 대화가 길어졌을 때 |
| `/exit` | 세션 종료 | 작업 완료 후 나갈 때 |
| `/quit` | 세션 종료 (`/exit`와 동일) | — |

### 프로젝트 설정

| 커맨드 | 동작 | 언제 사용하나 |
|--------|------|--------------|
| `/init` | 현재 프로젝트를 분석해 `CLAUDE.md` 초안 자동 생성 | 새 프로젝트 시작 시, CLAUDE.md를 처음 만들 때 |

### 정보·상태 확인

| 커맨드 | 동작 | 언제 사용하나 |
|--------|------|--------------|
| `/help` | 전체 커맨드 목록과 간단한 설명 출력 | 어떤 커맨드가 있는지 확인할 때 |
| `/cost` | 현재 세션의 토큰 사용량과 예상 비용 출력 | 비용이 궁금할 때, API 사용 중 과금 확인 시 |
| `/status` | 계정·API 연결 상태 확인 | 인증 문제가 의심될 때 |

### 설정·모드

| 커맨드 | 동작 | 언제 사용하나 |
|--------|------|--------------|
| `/model` | 현재 사용 중인 Claude 모델 확인 및 변경 | 속도·품질 트레이드오프가 필요할 때 |
| `/config` | 설정 파일 열기 또는 주요 설정 변경 | 권한, 동작 방식 등을 수정할 때 |

### 커스텀 커맨드 (5교시에서 실습)

프로젝트 루트의 `.claude/commands/` 디렉토리에 마크다운 파일을 만들면 나만의 슬래시 커맨드가 됩니다.

```
.claude/
└── commands/
    ├── review.md      → /review 커맨드
    └── commit.md      → /commit 커맨드
```

## CLI 플래그 — `claude` 실행 방법

터미널에서 `claude`를 어떻게 실행하느냐에 따라 동작 방식이 달라집니다.

### 기본 실행

```bash
claude                        # 대화형 모드 시작 (빈 세션)
claude "프롬프트 내용"          # 대화형 모드 + 첫 메시지 자동 입력
```

### 이전 대화 이어서

```bash
claude --continue             # 가장 최근 대화를 이어서 시작
claude -c                     # --continue 단축 옵션

claude --resume               # 대화 목록을 보여주고 선택해서 재개
claude -r                     # --resume 단축 옵션
```

### 비대화형(스크립트용)

```bash
claude --print "프롬프트"     # 응답만 출력하고 종료 (터미널 스크립트에 유용)
claude -p "프롬프트"          # --print 단축 옵션

# 예: 파일 설명을 텍스트로 저장
claude -p "auth.py를 한 문장으로 설명해줘" > description.txt
```

### 모델 지정

```bash
claude --model claude-opus-4-5      # 특정 모델로 시작
```

## 권한(Permission) 모드 상세

Claude Code는 세 가지 권한 수준으로 동작할 수 있습니다.

### 기본 모드 (Default)

파일 수정·명령 실행 전 매번 허락을 묻습니다. **처음 사용할 때 권장합니다.**

```
Claude Code가 다음을 실행하려고 합니다:
  $ pytest test_auth.py -v

허용하시겠습니까?  [예 / 아니오]
```

허용 선택지의 종류:

| 선택 | 동작 |
|------|------|
| 예 (이번만) | 이번 한 번만 허용 |
| 예, 앞으로도 이 파일 허용 | 같은 파일에 대해 이후 요청은 자동 승인 |
| 예, 앞으로도 이 명령 허용 | 같은 명령에 대해 이후 요청은 자동 승인 |
| 아니오 | 거부하고 에이전트가 다른 방법 찾음 |

### 자동 승인 모드

작업 중 반복되는 항목에 "앞으로도 허용"을 선택하면 누적됩니다. 설정 파일에서 사전 등록도 가능합니다.

### ESC — 즉시 중단

```
(작업 진행 중)  →  ESC 키  →  현재 작업 즉시 중단
```

에이전트가 잘못된 방향으로 가고 있다고 느끼면 언제든 ESC로 멈출 수 있습니다.

## 커맨드 사용 시나리오

### 시나리오 A: 새 작업 시작

```
이전 작업 종료
    ↓
/clear                     ← 이전 맥락 초기화
    ↓
새 요청 입력
```

### 시나리오 B: 대화가 길어졌을 때

```
작업 중 대화가 매우 길어짐
    ↓
/compact                   ← 대화를 요약해서 압축 (맥락은 유지)
    ↓
작업 계속
```

### 시나리오 C: 내일 이어서 작업

```
오늘 작업 종료
/exit
    ↓ (다음 날)
claude --continue          ← 이전 대화 그대로 이어서 시작
```

### 시나리오 D: 비용 확인 후 작업량 조절

```
작업 중
    ↓
/cost                      ← 토큰 사용량 확인
    ↓
비용이 많이 쌓였으면 /clear 후 핵심 내용만 다시 요청
```

## 빠른 참고표

```
/help      → 전체 커맨드 목록
/clear     → 새 작업 시작 (맥락 초기화)
/compact   → 대화 길어졌을 때 압축
/init      → CLAUDE.md 자동 생성
/cost      → 이번 세션 비용 확인
/exit      → 종료

claude --continue  → 이전 대화 이어서
claude --resume    → 대화 선택해서 재개
claude -p "..."    → 한 번만 실행하고 종료
ESC                → 작업 즉시 중단
```
