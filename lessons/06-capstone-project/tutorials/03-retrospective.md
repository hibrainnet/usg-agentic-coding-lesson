# 실습 3: 회고 (10분)

## 회고란?

작업이 끝난 뒤 "잘된 것"과 "개선할 것"을 돌아보는 짧은 점검입니다.  
에이전트 협업에서 회고는 특히 중요합니다 — 같은 실수를 반복하지 않고,  
자신에게 맞는 워크플로우를 빠르게 찾을 수 있습니다.

---

## 회고 질문지

다음 질문에 짧게 답해보세요. (노트에 적거나, Claude Code에서 대화로 진행)

### 잘 된 것

1. **가장 효과적이었던 프롬프트 패턴은?**
   - 어떤 요청 방식이 에이전트에서 좋은 결과를 냈나요?
   - 예: "테스트 먼저 작성하고 구현해줘", "범위를 명시한 요청"

2. **에이전트와 협업해서 혼자보다 빨라진 부분은?**
   - 어떤 작업에서 시간이 가장 절약됐나요?

3. **예상보다 잘 동작한 것은?**
   - 생각보다 에이전트가 잘 처리한 부분이 있었나요?

---

### 개선할 것

4. **에이전트가 실수한 순간은?**
   - 어떤 상황에서 수정이 필요했나요?
   - 다음번엔 어떻게 요청하면 나을 것 같나요?

5. **내가 실수한 프롬프트 패턴은?**
   - 모호하게 요청해서 원하는 결과가 나오지 않은 경우
   - 한 번에 너무 많이 요청해서 결과가 엉킨 경우

6. **다음에 꼭 바꿔볼 것 한 가지는?**
   - 구체적으로: "다음에는 ~~하겠다"

---

## 에이전트와 함께 회고하기

Claude Code에서 직접 회고를 진행할 수도 있습니다.

```
"오늘 이 프로젝트를 진행하면서 나눈 대화를 돌아보고,
내가 사용한 프롬프트 패턴 중 잘 동작한 것 3가지와
개선이 필요한 것 2가지를 알려줘."
```

```
"오늘 내가 에이전트에게 요청한 방식 중에서
더 명확하게 요청할 수 있었던 케이스를 2개 찾아줘.
어떻게 바꾸면 더 좋았을지도 알려줘."
```

---

## 나만의 워크플로우 카드 만들기

회고를 바탕으로 자신만의 협업 워크플로우를 정리합니다.

```
"오늘 배운 것을 바탕으로 나만의 에이전트 협업 워크플로우를
5단계로 정리해줘. 각 단계에 핵심 팁을 한 줄씩 추가해줘."
```

이 카드는 다음 프로젝트에서 바로 참고할 수 있는 치트시트가 됩니다.

## 강의 전체 회고

6교시 전체를 마무리하면서 큰 그림을 돌아봅니다.

| 교시 | 핵심 도구 | 내가 가장 유용하게 쓸 것 |
|------|-----------|--------------------------|
| 1교시 | Agentic Coding 개념 | |
| 2교시 | Claude Code 기본 워크플로우 | |
| 3교시 | CLAUDE.md, 프롬프팅 | |
| 4교시 | TDD, 디버깅, Git | |
| 5교시 | 커스텀 커맨드, 훅, MCP | |
| 6교시 | 전체 통합 + 베스트 프랙티스 | |

위 표를 채우고, 실제 프로젝트에서 가장 먼저 적용할 것 1가지를 결정합니다.
