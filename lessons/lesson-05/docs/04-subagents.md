# 서브에이전트: 탐색 작업 위임

## 서브에이전트란?

메인 에이전트가 **독립적인 하위 에이전트를 생성해 작업을 분리**하는 패턴입니다.  
긴 탐색 작업이나 독립적인 분석을 서브에이전트에게 맡기면, 메인 컨텍스트를 깔끔하게 유지할 수 있습니다.

```
메인 에이전트  →  서브에이전트  →  탐색/분석 결과
(컨텍스트 보존)   (독립 컨텍스트)   (요약만 반환)
```

---

## 서브에이전트가 필요한 상황

```
✅ "task-manager의 모든 파일을 읽고 아키텍처를 분석해줘"
✅ "lessons/ 전체에서 테스트가 없는 파이썬 파일을 찾아줘"
✅ "50개 파일을 모두 읽고 의존성 맵을 그려줘"
```

이런 작업을 메인 대화에서 직접 하면 컨텍스트가 금방 차서 이후 대화가 느려집니다.  
서브에이전트에게 위임하면 결과 요약만 메인 대화로 돌아옵니다.

---

## 서브에이전트 호출 방법

프롬프트에 "서브에이전트를 사용해서" 또는 "독립 에이전트에게 맡겨줘"라고 명시합니다.

```
"서브에이전트를 사용해서 탐색해줘:
 lessons/lesson-05/examples/task-manager/ 디렉토리를 분석하고
 구현된 함수 목록과 각 함수에 테스트가 있는지 정리해줘.
 탐색은 독립 에이전트에게 맡기고 결과만 요약해서 알려줘."
```

---

## 스킬의 context: fork

스킬의 `context: fork` frontmatter를 사용하면, 스킬 자체가 서브에이전트에서 실행됩니다.

```yaml
---
description: task-manager 코드베이스를 심층 분석한다
context: fork
agent: Explore
---

lessons/lesson-05/examples/task-manager/ 디렉토리를 분석해줘:
1. 구현된 함수 목록과 역할
2. 테스트가 없는 함수 파악
3. 개선 제안
```

`agent: Explore`는 파일 탐색에 최적화된 읽기 전용 에이전트입니다.

## 직접 탐색 vs 서브에이전트 비교

| | 직접 탐색 | 서브에이전트 위임 |
|--|--|--|
| **컨텍스트 소비** | 메인 대화에 모두 쌓임 | 서브에이전트에서 처리 후 요약만 반환 |
| **적합한 작업** | 단순하고 짧은 탐색 | 넓고 긴 탐색 (파일 수십 개 이상) |
| **결과 형식** | 전체 과정이 대화에 노출 | 요약된 결과만 메인 대화에 등장 |
| **속도** | 즉시 시작 | 에이전트 생성 오버헤드 있음 |

## 스킬 · 훅 · MCP · 서브에이전트 비교

| | 스킬 | 훅 | MCP | 서브에이전트 |
|--|--|--|--|--|
| **누가 실행하나** | 사람이 `/스킬명`으로 호출 또는 Claude 자동 로드 | 이벤트 발생 시 자동 | 에이전트가 요청 시 | 에이전트가 위임 |
| **무엇을 하나** | 반복 프롬프트·절차 저장 | 쉘 명령어 자동 실행 | 외부 도구 접근 | 독립 탐색/분석 |
| **주요 사용처** | 리뷰, 테스트, 커밋 | 포맷, 알림, 자동 테스트 | DB, GitHub, 파일 | 대용량 탐색 |
| **파일 위치** | `.claude/skills/<이름>/SKILL.md` | `.claude/settings.json` | `settings.json` mcpServers | 프롬프트로 위임 |

## 핵심 정리

```
✅ 서브에이전트 = 독립 컨텍스트에서 실행되는 하위 에이전트
✅ 넓은 탐색·분석 작업에 사용
✅ 메인 대화에는 결과 요약만 반환
✅ 스킬의 context: fork 로 스킬 자체를 서브에이전트로 실행 가능
```
