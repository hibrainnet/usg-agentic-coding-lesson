---
description: task-manager 변경 사항에서 버그·엣지 케이스·테스트 누락을 찾아 보고한다
---

`git diff HEAD`를 분석해서 task-manager 코드 리뷰를 해줘.

다음 네 가지 항목을 각각 확인해줘:

**1. 버그 & 논리 오류**
- task id 할당 로직, 상태(done) 전환, 리스트 변경 부작용 등
- 발견하면 파일명:줄번호와 함께 구체적으로 설명해줘

**2. 엣지 케이스 누락**
- 빈 제목, None, 존재하지 않는 ID, 빈 리스트 등 처리가 빠진 케이스
- 각 케이스에서 어떤 문제가 생기는지 설명해줘

**3. 테스트 커버리지**
- 새로 추가된 함수나 분기에 테스트가 있는지 확인해줘
- 테스트가 없으면 어떤 케이스를 테스트해야 하는지 제안해줘

**4. 코드 가독성**
- 함수·변수명이 명확한지
- 이해하기 어려운 로직이 있는지

변경 사항이 없으면 "변경 사항이 없습니다."라고 알려줘.
발견된 문제가 없으면 "리뷰 결과: 문제 없음"이라고 알려줘.
